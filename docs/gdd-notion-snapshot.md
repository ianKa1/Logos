# 石头剪刀布卡牌对战 游戏设计文档（GDD）


## 设计支柱（First Principles）


以下为项目长期稳定的设计支柱，所有设计内容须与之保持一致：

1. **稳定的数值系统框架**：建立攻击、防御、资源之间的统一换算公式（替换公式），作为所有卡牌数值设计的共同基础。**Combo 原则**：多卡组合的总数值收益应超过等量单卡收益，鼓励组合构筑而非单卡堆数值。
2. **核心新玩法：玩家自创卡牌**：玩家自行设计卡牌，借助 AI 落地到游戏内。分两类定位——**对抗向卡牌**（新机制、新数值，重策略对抗）与**个人元素卡牌**（融入玩家个人元素如宠物，重有趣恶搞）。
3. **社交元素**：**好友对战**（瞎搞娱乐，规则宽松、包容恶搞卡牌）与**天梯对战**（认真构筑、追求胜利或 combo 爽快感，对数值平衡要求高）。
4. **收集元素**：卡牌有稀有度区分，玩家拥有自己的牌库；不同稀有度在效果或特效上有明显差别；提供精美的卡牌收藏体验。

## 1. 游戏概述

- **类型**：1v1 对战卡牌游戏。
- **核心创新点**：玩家在对局中通过类似伪代码的界面**自行创造卡牌效果**，由 AI 实时判定语句合法性并计算行动点消耗。（服务于支柱 2：玩家自创卡牌）
- **当前状态说明**：核心对战玩法目前有**两个竞争中的候选方案**，均未标记【决定】：
    - **方案A-石头剪刀布对战**：以石头剪刀布为判定核心，规则相对完整，完整流程见其子文档。
    - **方案B-回合制+随从+玩家**：采用最经典的回合制结构，目前草稿仅给出核心结构的简要描述，细节尚待补充。

    二者是核心玩法层面的竞争关系，将并列呈现、对比，不合并为一个设计，直至其中一个被标记【决定】。

## 3. 卡牌创造系统


以下为卡牌创造系统的通用设计（当前依托**方案A**的行动点体系描述，方案A 内的详细流程与定价示例见其子文档；**方案B** 尚未定义其卡牌创造系统如何嵌入，见 Open Questions）：

- **界面**：类伪代码编辑器，逐句输入。
- **基础素材**：系统提供基础语句与关键字（如果…、胜点、上回合、本回合、下回合、透视等），玩家在此基础上自由撰写。
- **AI 判定**：每输入一句，AI 实时判定合法性，并汇总给出整张卡牌的行动点消耗。
- **定价逻辑（通用原则）**：
    - 信息类效果（如透视）有基础消耗。
    - 基于已有信息的条件效果消耗较高；基于预测的条件效果消耗较低。
    - 胜点增减幅度越大，消耗越高，可能呈指数增长。
    - 对自己施加负面效果可以反向获得行动点。
- 与支柱 1 的一致性说明：单效果数值收益的指数级消耗增长天然抑制"单卡堆数值"，与 Combo 原则（组合收益 > 等量单卡收益）方向一致；具体如何在行动点定价中体现 combo 加成，见 Open Questions。

# 方案A：石头剪刀布对战（详细设计）


## 1. 概述与胜利条件


核心机制基于石头剪刀布。**胜利条件**：1v1 进行若干轮石头剪刀布，一方**胜点**累积到目标值（暂定 13，待定）时获胜。


## 2. 核心循环


每回合按以下顺序进行四个流程：

1. **放置流程 1** → 2. **创造流程** → 3. **放置流程 2** → 4. **结算流程**

重复回合，直到一方胜点达到目标值。


## 3. 行动点系统

- 双方初始各有行动点（暂定 2，待定）。
- 行动点可在**创造流程**和**放置流程 2** 中消耗。
- 未用完的行动点**继承到下回合**。
- 每回合自然补充行动点（暂定 2，待定）。
- 行动点**可能设有上限**（待定）。

## 4. 流程详细规则


### 4.1 放置流程 1

- 双方各自秘密放置石头、剪刀、布中的一个手势。

### 4.2 创造流程

- 玩家通过游戏提供的**类似伪代码的界面**设计一张卡牌。
- 游戏提供基础语句和关键字，例如：**如果…、胜点、上回合、本回合、下回合、透视** 等。
- 在基础语句和关键字之上，玩家可以**自己打字定义效果**。
- 每打一句话，AI 会判断该句是否合法，并给出该卡牌**总共的行动点消耗**。
- 创造流程有时间限制（暂定 1 分钟，待定）。
- 每次创造流程开始时，玩家可选择：
    - 创造一张**新的**卡牌；或
    - 在一张**之前创造了但没有打出过**的卡牌上继续创造。
- 卡牌创造结束后，玩家可以**打出一张已经创造的卡牌**，并消耗该卡牌对应的行动点；消耗值不能大于玩家当前拥有的行动点。

### 效果消耗设计原则


| 示例                            | 效果             | 消耗规律                                  |
| ----------------------------- | -------------- | ------------------------------------- |
| 示例 1：透视                       | 看到对方在放置流程放置的手势 | 消耗暂定 1 行动点（待定）                        |
| 示例 2：如果（对方上回合出了布）我方胜点 +1      | 基于**已发生**情况做判断 | 消耗相对较多行动点                             |
| 示例 2 变体：预测对方**本回合**或**下回合**出布 | 基于**未发生**情况做预测 | 消耗相对较少行动点                             |
| 示例 3：对方胜点 -10                 | 直接改变胜点         | 对方胜点减少量 / 我方胜点增加量越大，消耗越多，且可能是**指数相关** |
| 示例 4：敌方透视                     | 对自己的**负面效果**   | 可以**增加**自己的行动点                        |


### 4.3 放置流程 2

- 开始时，**结算已放置卡牌的效果**。
- 之后玩家可以消耗行动点（暂定 1，待定）**更改自己的手势**。

### 4.4 结算流程

- 双方亮出手势，进行石头剪刀布判定。
- **赢的一方胜点 +1**。
