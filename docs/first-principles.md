# 设计第一性原则（First Principles）


本页定义项目的设计支柱，原则上长期稳定、极少改动。所有草稿与 GDD 内容都应与本页保持一致。


## 1. 稳定的数值系统框架

- 建立攻击、防御、资源之间的统一换算公式（替换公式），作为所有卡牌数值设计的共同基础。
- **Combo 原则**：多张卡牌组成的 combo，其总数值收益应超过等量单张卡牌的数值收益——鼓励组合构筑，而非单卡堆数值。

## 2. 核心新玩法：玩家自创卡牌


玩家自己设计卡牌，借助 AI 的能力将卡牌落地到游戏内。自创卡牌分为两类定位：

- **对抗向卡牌**：承载新机制、新数值，重在策略与对抗。
- **个人元素卡牌**：融入玩家的个人元素（如自己的宠物），重在有趣与恶搞。

## 3. 社交元素


| 模式   | 玩家体验定位                  | 设计要求        |
| ---- | ----------------------- | ----------- |
| 好友对战 | 瞎搞、怎么恶心怎么来，纯娱乐          | 规则宽松，包容恶搞卡牌 |
| 天梯对战 | 认真构筑牌组，追求胜利或 combo 的爽快感 | 对数值平衡性要求较高  |


## 4. 收集元素

- 卡牌具有稀有度区分；玩家拥有自己的牌库。
- 不同稀有度的卡牌在**效果或特效**上有明显差别。
- 提供精美的卡牌收藏体验。
